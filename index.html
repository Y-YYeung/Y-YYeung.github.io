Objective-C 高级编程 Chapter 1

By Wen Yongyang

發表於 2016-02-01

内存管理思考方式

自己：对象的使用环境

- 自己生成的对象，自己持有
- 非自己生成的对象，自己也能持有
- 不再需要自己持有的对象时释放
- 非自己持有的对象无法释放

  • 对象操作及其对应方法

    • 生成并持有 | alloc/new/copy/mutableCopy
    • 持有 | retain
    • 释放 | release
    • 废弃 | dealloc

autorelease

- 使用方法
    - 生成并持有 NSAutoreleasePool 对象
    - 调用已分配对象的 autorelease 实例方法
    - 废弃 NSAuroreleasePool 对象 （对对象逐一调用 release 方法）

- NSAutoreleasePool 生命周期相当于 C 语言的变量作用域 （NSRunLoop 的一次周期）

- 可手动嵌入多个 NSAutoreleasePool，如在图像处理过程中

- 可通过 NSAutoreleasePool 中调试用的非公开类方法 `showPools` 确认已被 autorelease 的对象状况 (仅适用于 iOS)
- 也可使用 `_objc_autoreleasePoolPrint`

ARC 规则

  • 一个项目中，可以以文件为单位指定 ARC 有效或无效

  • ARC 式的内存管理属于编译器的工作

  • 设置 ARC 有效的方法

    • 使用 clang(LLVM编译器) 3.0 或以上版本
    • 指定编译器属性为 “-fobjc-arc”
    • Xcode 4.2 默认对所有文件为 ARC 有效

  • ARC 有效时的所有权修饰符

    • __strong
    • __weak
    • __unsafe_unretained
    • __autorelease

  • __strong

    • id 类型和对象类型默认

  • __unsafe_unretained

    • 该变量不属于编译器的内存管理对象
    • 作用与 __weak 一样
    • 对象指针不会自动置 nil，会成为悬垂指针
    • 适配 iOS4 和 OS X Snow Leopard

  • __autorelease
    使用区块

- 编译器会检查方法名是否以 alloc/new/copy/mutableCopy 开始，若不是，自动将返回值的对象注册到 autoreleasepool

Run Loop

By Wen Yongyang

發表於 2016-01-12

RunLoop

关于 RunLoop，系统提供的对象:

  1. CFRunLoopRef 位于 CoreFoundation 框架内，提供纯 C 函数 API，所有 API 线程安全
  2. NSRunLoop 基于 CFRunLoopRef 的封装，提供面向对象的 API，但线程不安全

关于线程，重要的对象：

  1. pthread_t
  2. NSThread

Apple 不允许直接创建 RunLoop，提供自动获取的函数

  1. CFRunLoopGetMain()
  2. CFRunLoopGetCurrent()

线程与 RunLoop 一一对应，关系保存在一个全局 Dictionary
线程刚创建时没有 RunLoop，该 RunLoop 需主动获取才有

主线程的 RunLoop 两个预置的 Mode，均被标记为 Common(后面用到):

  1. kCFRunLoopDefaultMode App 平时所处的状态
  2. UITrackingRunLoopMode 追踪 Scroll View 滑动时的状态

当创建一个 Timer 添加到 DefaultMode 时，Timer 会重复调用，但滑动一个 Table View 时，RunLoop 切换到 Tracking Mode，暂停 Timer 调用

解决方法，在两个 Mode 中得到回调：
方法1：将 Timer 加入两个 Mode 中
方法2：将 Timer 加入到顶层 RunLoop 的 commonModeItems，commonModeItems 会被 RunLoop 自动更新到具有 Common 属性的 Mode 中，即上述的两种 Mode

空指针 / 空对象

By Wen Yongyang

發表於 2016-01-12

nil / Nil / NULL / NSNull

在 C 语言中，用 0 表示 不存在 的原始值，NULL 作为指针 不存在 的原始值

Objective-C 在 C 的基础上增加 nil，nil 指向不存在的对象指针，与 NULL 技术上相等

Foundation 定义了 NSNull，只有一个类方法 + null，单例模式，返回一个 NSNull 对象，是一个实际的对象，与 nil, NULL 不同

The NSNull class defines a singleton object used to represent null values in collection objects (which don’t allow nil values).

Something about nil

  • 刚被分配的 NSObject 内容被设置为 0，该对象的所有指针都从 nil 开始
  • 向 nil 发送消息，返回 0 值，避免在使用 nil 前进行检查

Something about NSNull

  • 解决如NSArray和NSDictionary之类的集合不能有nil值的缺陷
  • 可以将NSNull理解为有效的将 NULL 或者 nil 值封装 boxing
