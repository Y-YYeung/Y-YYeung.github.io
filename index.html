1. 使用 CAShapeLayer 对视图进行绘制，可以参考 这个 gitbooks

优点是：

• 渲染快速。CAShapeLayer使用了硬件加速，绘制同一图形会比用Core Graphics快很多。
• 高效使用内存。一个CAShapeLayer不需要像普通CALayer一样创建一个寄宿图形，所以无论有多大，都不会占用太多的内存。
• 不会被图层边界剪裁掉。一个CAShapeLayer可以在边界之外绘制。你的图层路径不会像在使用Core Graphics的普通CALayer一样被剪裁掉（如我们在第二章所见）。
• 不会出现像素化。当你给CAShapeLayer做3D变换时，它不像一个有寄宿图的普通图层一样变得像素化。

SQL 基本语句

SQL 执行过程须知：

  • 格式：

  • 每个步骤都会产生一个虚拟表，用作下一个步骤的输入，且这些虚拟表不对外开放

  • 若没有在查询中指定某一子句，将跳过相应的步骤

  • 执行顺序：

    1. FROM 对 FROM 子句中的前两个表执行笛卡尔积，-> 虚拟表 VT1
    2. ON 对 VT1 应用 ON 筛选器，-> VT2
    3. OUTER (JOIN) -> VT3
    4. WHERE 对 VT3 应用筛选器， == true -> VT4
    5. GROUP BY 对 VT4 进行 行分组， -> VT5
    6. CUBE | ROLLUP 把超组添加到 VT5, -> VT6
    7. HAVING 对 VT6 应用筛选器， == true -> VT7
    8. SELECT -> VT8
    9. DISTINCT 去除 VT8 中的重复行 -> VT9
    10. ORDER BY 对 VT9 按 进行排序，-> 游标 arrow
    11. TOP 根据 arrow 选择数量的行，-> VT10, 结果返回

With 语句

  • with 语句需要与 Context Manager 一起
  • with 语句规范一个动作进入与离开某个作用域是自动进行的一些动作
  • 如读写文件前后的打开或关闭文件操作，使用 with 语句就可以使得这两个动作自动进行，而不需要使用者繁琐地进行代码控制
  • with 语句自动调用 __enter__() 和 __exit__() 两个方法，因此，若我们需要为自己的类设置自动动作，则重写以上两个方法
  • __enter__() 返回一个供 with 语句使用的对象，通常也和 as 一起使用，如：
  • __exit__(self, exc_type, excc_value, traceback) 三个参数分别为：异常类型，异常值，traceback，若执行 with 语句过程中没有产生异常，则3个参数都为 None，否则，则将3个参数的值返回给 __exit__() 使用

if __name__ == __main__

By Wen Yongyang

发表于 2016-03-25

if __name__ == __main__

  • Python 文件的后缀名为 .py
  • .py 文件可直接执行，可作为（功能）模块被导入
  • __name__ 是模块的内置属性，代表 .py 文件的调用方式
  • __main__ 表示 .py 文件是直接执行的
  • 综上，if __name__ == __main__ 表示：当前 .py 文件是作为直接调用的方式执行，而不是作为模块被调度执行

下面是来自官方文档对 __main__ 的解释

‘main‘ is the name of the scope in which top-level code executes. A module’s name is set equal to ‘main‘ when read from standard input, a script, or from an interactive prompt.

A module can discover whether or not it is running in the main scope by checking its own name, which allows a common idiom for conditionally executing code in a module when it is run as a script or with python -m but not when it is imported:

For a package, the same effect can be achieved by including a main.py module, the contents of which will be executed when the module is run with -m.

下面是来自官方文档对 __name__ 的解释

The name attribute must be set to the fully-qualified name of the module. This name is used to uniquely identify the module in the import system.

内存管理思考方式

自己：对象的使用环境

- 自己生成的对象，自己持有
- 非自己生成的对象，自己也能持有
- 不再需要自己持有的对象时释放
- 非自己持有的对象无法释放

  • 对象操作及其对应方法

    • 生成并持有 | alloc/new/copy/mutableCopy
    • 持有 | retain
    • 释放 | release
    • 废弃 | dealloc

autorelease

- 使用方法
    - 生成并持有 NSAutoreleasePool 对象
    - 调用已分配对象的 autorelease 实例方法
    - 废弃 NSAuroreleasePool 对象 （对对象逐一调用 release 方法）

- NSAutoreleasePool 生命周期相当于 C 语言的变量作用域 （NSRunLoop 的一次周期）

- 可手动嵌入多个 NSAutoreleasePool，如在图像处理过程中

- 可通过 NSAutoreleasePool 中调试用的非公开类方法 `showPools` 确认已被 autorelease 的对象状况 (仅适用于 iOS)
- 也可使用 `_objc_autoreleasePoolPrint`

ARC 规则

  • 一个项目中，可以以文件为单位指定 ARC 有效或无效

  • ARC 式的内存管理属于编译器的工作

  • 设置 ARC 有效的方法

    • 使用 clang(LLVM编译器) 3.0 或以上版本
    • 指定编译器属性为 “-fobjc-arc”
    • Xcode 4.2 默认对所有文件为 ARC 有效

  • ARC 有效时的所有权修饰符

    • __strong
    • __weak
    • __unsafe_unretained
    • __autorelease

  • __strong

    • id 类型和对象类型默认

  • __unsafe_unretained

    • 该变量不属于编译器的内存管理对象
    • 作用与 __weak 一样
    • 对象指针不会自动置 nil，会成为悬垂指针
    • 适配 iOS4 和 OS X Snow Leopard

  • __autorelease
    使用区块

- 编译器会检查方法名是否以 alloc/new/copy/mutableCopy 开始，若不是，自动将返回值的对象注册到 autoreleasepool
