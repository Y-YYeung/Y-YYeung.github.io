1. 使用 CAShapeLayer 对视图进行绘制，可以参考 这个 gitbooks

优点是：

• 渲染快速。CAShapeLayer使用了硬件加速，绘制同一图形会比用Core Graphics快很多。
• 高效使用内存。一个CAShapeLayer不需要像普通CALayer一样创建一个寄宿图形，所以无论有多大，都不会占用太多的内存。
• 不会被图层边界剪裁掉。一个CAShapeLayer可以在边界之外绘制。你的图层路径不会像在使用Core Graphics的普通CALayer一样被剪裁掉（如我们在第二章所见）。
• 不会出现像素化。当你给CAShapeLayer做3D变换时，它不像一个有寄宿图的普通图层一样变得像素化。

3. 关于 NSTimer

  • NSTimer 实例会 retain target
  • 尽量指定 NSTimer 实例运行的 RunLoop, 特别与 UITableView 等控件一起用时
  • 暂停 NSTimer, 综合各种方法，其实就是把 NSTimer 实例杀了

4. 关于 NSMutableArray 的 KVC, 可以查看 stackoverflow

得票最高的答案大致是：
由于其方法以及内容可变的原因， NSMutableArray 并不能进行 KVC。而解决方案是，使用 proxying 代理来进行（这有待进一步学习，代理模式），当然，设计继承于 NSMutableArray 也可以，但使用 proxying 可以更加简单。

NSProxy 是一个简单而强大的类。我们可以使用它来拦截那些发送到数组的消息，就像这个类就是数组自己一样。然后，我们再将此条信息传递给内部的数组。然而，这还是不能使用 KVC。但由于 KVC 是 NSObject 的一个特点，所以，我们可以利用这一点。（所以，这里应该还是利用的代理模式的思想罢了）
