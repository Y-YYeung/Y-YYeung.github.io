建立硬盘分区)

  • genw划分多个逻辑区块，区块间的访问相互独立
  • 可从逻辑上隔离数据
  • 每个分区使用前需格式化为文件系统

分区表

  • 分区信息存放地点
  • 主流模式：
    • 传统：Master Boot Record (MBR)
    • 新型：GUID Partition Table (GPT)

Master Boot Record

  • 最多支持4个分区
  • 利用扩展分区和逻辑分区绕过上述限制
  • 分区类型
    • 主分区 (Primary)
    • 扩展分区 (Extended)
      • 逻辑分区 (Logical)
      • 扩展分区 可看作 逻辑分区 的容器
      • 扩展分区包含的逻辑分区没有限制
  • 通常习惯
    • 创建主分区 sda1 到 sda3，建立一个扩展分区 sda4
    • sda4 中包含 sda5，sda6 等逻辑分区

GPT

  • 只有一种分区类型：主分区

必须使用 MBR 情况

  • 使用 GRUB legacy 作为 bootloader
  • 使用传统的 BIOS，且双启动中包含 Windows(32/64)

必须使用 GPT

  • 使用 UEFI 而不是 BIOS，且双启动中包含 Windows 64

分区方案

  • 单 root 分区
  • 多分区
    • 将某个路径挂载为独立分区，使其拥有不同的文件系统和挂在参数，可被多个操作系统共享

挂载点

  • / (根分区)
    • 目录树顶层
    • 主文件系统挂载和其它文件系统挂载的地方
    • 其中的内容应满足于：
      • 启动
      • 恢复
      • 修复系统
    • 其下特定目录不能作为独立分区
  • /boot
    • 包含
      • 内核
      • ramdisk 镜像
      • bootloader 配置文件
      • bootloader stage
    • 内核在使用用户态程序前使用的其它数据
    • 日常系统运行并不需要，只在启动，内核升级（重建 initial ramdisk）时用到
  • /home
    • 包含
      • 用户定义的配置文件、缓存、应用程序数据、媒体文件
      • 可在不修改 /home 内容情况下重装 Arch，删除其它顶级目录，执行 pacstrap
  • /var
    • 存储：
      • 变量数据，如 spool 目录和文件
      • 管理和登录数据
      • pacman 的缓存
      • ABS 树。。。。
    • 常用做 缓存或日志记录，读写频繁
    • 独立作用：避免由于大量日志写入造成磁盘空间耗尽
  • /tmp
    • 默认情况下，已经是独立分区
  • Swap
    • 虚拟内存空间

分区大小设置

  • /boot
    • 200MB
    • 实际需要约 100MB
  • /
    • 15-20GB
    • 传统上包括 /usr 目录
  • /var
    • 8-12GB
  • /home
    • 不定
    • 在桌面系统中，通常是最大的文件系统
  • Swap
    • 不定，通常是物理内存的2倍
