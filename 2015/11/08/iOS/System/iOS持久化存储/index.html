应用程序包

  • 存放应用程序源文件，含资源文件，可执行文件

Documents

  • 常用目录，iTunes 同步，适合存放重要数据

Library / Caches

  • iTunes 不会同步，适合存放体积大，不需要备份的非重要数据

Library / Preferences

  • iTunes 同步，保存应用的设置信息

tmp

  • iTunes 不会同步，系统在应用不运行时删除该目录下的文件

plist 文件

  • 将某些特定的类，通过XML文件方式保存在目录中

  • 获取文件路径

  • 存储

  • atomically: 是否需要先写入一个辅助文件，再把辅助文件拷贝到目标文件地址，安全写入文件的做法

  • 读取

  • 就是使用 arrayWithContentsOfFile: 来读取！

Preference

  • 专门用来保存应用程序的配置信息，最好不要保存其他数据

  • 会将所有数据保存到同一个文件中 -> preference 目录下一个以此应用包命名的 plist 文件

  • 获取 NSUserDefaults 文件

  • 向文件中写入内容

  • 立即同步

  • 读取文件

NSKeyedArchiver

  • 归档 -> 遵循 NSCoding 协议的对象可以通过它来实现 序列化

  • 大多数指出数据存储的 Foundation 和 Cocoa Touch 类都遵循来 NSCoding 协议

  • 遵循协议和设置属性

  • 实现协议方法

  • 将对象归档 archiveRootObject: toFile:

  • 从文件中解档对象 unarchiveObjectWithFile:

  • 保存的文件扩展名可随意

  • 继承时必须先调用父类的归档解档方法
